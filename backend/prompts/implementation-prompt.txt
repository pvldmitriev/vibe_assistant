ВАЖНО: WORKFLOW ДЛЯ ПРАВИЛ CURSOR
Cursor не может напрямую редактировать .mdc файлы в .cursor/rules/

СОЗДАНИЕ: .txt → редактировать → .mdc → переместить в .cursor/rules/
РЕДАКТИРОВАНИЕ: скопировать из .cursor/rules/ → .txt → редактировать → .mdc → вернуть

CURSOR ИНСТРУМЕНТЫ:
- create_plan - создание плана с todos (Ctrl+Shift+P → "Plan")
- todo_write - управление todos (отмечай выполненные задачи!)
- Composer - режим для больших задач (Ctrl+Shift+P → "Composer: Open")

УРОВЕНЬ КАЧЕСТВА КОДА В ЗАВИСИМОСТИ ОТ ЦЕЛИ:

{{#if goal == "Обучение и практика"}}
ПОДХОД ДЛЯ ОБУЧЕНИЯ:
- Фокус на обучении и понимании, не на идеальном коде
- Можно использовать простые решения даже если они не оптимальны
- Можно использовать "костыли" если они помогают понять концепцию
- Главное чтобы работало и было понятно КАК работает
- Комментарии с объяснениями приветствуются
- НЕ пиши тесты (если только сам не хочешь попрактиковаться)

{{#if goal == "Использовать самому"}}
ПОДХОД ДЛЯ ЛИЧНОГО ИСПОЛЬЗОВАНИЯ:
- Рабочий надежный код
- Не делай overengineering - выбирай простые решения
- Код должен быть достаточно читаемым чтобы через месяц понять что к чему
- Ошибки обрабатывай там где они критичны
- Тесты пиши только если считаешь нужным
{{/if}}

{{#if goal == "Для пользователей"}}
ПОДХОД ДЛЯ ПРОДАКШН:
- Best practices во всем
- Security first - валидация всех входных данных
- Performance важен - следи за оптимизацией
- Обработка ВСЕХ edge cases и ошибок
- Понятные сообщения об ошибках для пользователей
- Логирование ошибок для отладки
- Код готовый к масштабированию
- ОБЯЗАТЕЛЬНО пиши тесты как указано в плане
{{/if}}

{{#if goal == "Портфолио"}}
ПОДХОД ДЛЯ ПОРТФОЛИО:
- Идеальный код как из учебника
- Design patterns где они уместны (не злоупотребляй!)
- Полная типизация (TypeScript если возможно)
- Подробные комментарии демонстрирующие твое понимание
- Код должен впечатлять - представь что его читает senior разработчик
- SOLID принципы, чистая архитектура
- ОБЯЗАТЕЛЬНО высокое покрытие тестами

{{/if}}

===

ИНСТРУКЦИИ ПО РЕАЛИЗАЦИИ:

1. Следуй плану и todos
   Выполняй задачи по порядку, не прыгай между задачами

2. Отмечай завершенные задачи
   Используй todo_write чтобы отметить задачу как "completed"
   Это важно для отслеживания прогресса!

3. Соблюдай правила из .cursor/rules/
   Они были созданы специально для этого проекта
   Следуй структуре папок и стандартам кода

4. {{#if goal == "Для пользователей" || goal == "Портфолио"}}
   Пиши тесты как указано в плане
   Не пропускай todos на тестирование!
   {{/if}}

5. Используй Composer для больших задач
   Если задача сложная или затрагивает много файлов - используй Composer mode

6. НЕ ОСТАНАВЛИВАЙСЯ пока все todos не завершены
   Продолжай работу до полного выполнения плана
   Если застрял - попроси помощи, но не останавливайся просто так

===

ПОМНИ О ПРАВИЛАХ .MDC:

.cursor/rules/file.mdc → копируй → file.txt → редактируй → file.mdc → верни обратно
