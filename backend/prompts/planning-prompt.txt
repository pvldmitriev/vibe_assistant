ТРЕБОВАНИЯ К ПЛАНУ:

1. Разбей разработку на конкретные задачи (todos)
   Каждая задача - это один логический блок работы
   Размер: 1-3 часа работы (не больше!)

2. Определи зависимости между задачами
   Что нужно сделать сначала, что потом
   Например: модели данных → API endpoints → фронтенд компоненты

3. Приоритизируй правильно
   Сначала: базовая структура и core функции
   Потом: дополнительные фичи и улучшения
   {{#if goal == "Для пользователей" || goal == "Портфолио"}}
   В конце: тесты, документация, полировка
   {{/if}}

4. Для каждой задачи укажи критерии готовности
   Как понять что задача выполнена?
   Что должно работать?

ИСПОЛЬЗУЙ ИНСТРУМЕНТ create_plan:

Вызови create_plan с массивом todos:
[
  {
    id: "unique-id-1",
    content: "Описание задачи",
    status: "pending"
  },
  ...
]

Статусы: "pending", "in_progress", "completed", "cancelled"

ВАЖНО: Первая задача должна быть самой приоритетной!
